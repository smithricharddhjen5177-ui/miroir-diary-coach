# Miroir — 日记教练

> 把"写日记"变成"聊日记"——聊完，日记就写好了。

## 为什么做这个项目

每个人都听说过"写日记改变人生"。但现实是：大多数人只会在极度痛苦的时候打开日记本，写满几页情绪，然后合上，下次打开又是几个月后——同样因为痛苦。

更糟糕的是，有时候"该写日记了"这件事本身也成了一种负担。面对空白页面，不知道该写什么、怎么写、写多少算够。于是干脆不写。

**问题不在于写日记没有用，而在于启动成本太高了。** 坐下来、打开本子、组织语言、写出有条理有逻辑的文字——这整个过程对很多人来说已经足够劝退。

Miroir 试图回答一个问题：**如果写日记的成本低到和聊天一样，会发生什么？**

你不用写出精致的句子，不用有逻辑、有条理。你只需要把散落的想法倒出来——就像和朋友聊今天发生了什么、心情怎么样。AI 会把这些碎片整理成结构化的日记。

核心想法很简单：

- **聊，而不是写** —— 和 AI 聊一聊你今天做了什么、经历了什么，启动成本远低于坐下来"好好写日记"
- **AI 帮你结构化** —— 散乱的表达交给 AI 整理，你只负责表达
- **想自己写也可以** —— AI 是可选的。不想用 AI 的时候，自由书写始终在那里

未来的方向：AI 不仅仅帮你整理，还能识别你思维中的盲点和反复出现的模式——但这需要更多训练数据，目前只是起步。

## 核心原则

- **启动成本比完美更重要** — 只写 3 行、只说 2 分钟，都算完成
- **感受先于事实** — 日记不只是记录"发生了什么"，更重要的是"我感受到什么"
- **完成胜于完美** — 先交一版上去，哪怕是粗糙的
- **下一个十五分钟** — 面对巨大恐惧时，不问"这辈子怎么办"，只问"下一个 15 分钟做什么"
- **两件事可以同时成立** — 想休息和想逃避不矛盾，诚实和温柔也不矛盾

## 当前状态：做到了什么，还差什么

### ✅ 已达成

| 目标 | 状态 |
|---|---|
| AI 教练对话 —— 和 AI 聊今天发生了什么 | ✅ `/coach` 页面，流式对话，引导式追问 |
| AI 晚间复盘 —— 帮用户把碎片想法结构化 | ✅ AI 读日记内容，生成事实摘要+发现+下一步 |
| 自由书写 —— 不想用 AI 时自己写 | ✅ TipTap 编辑器，不限格式 |
| 本地 Markdown 存储 —— 数据自有 | ✅ `content/journal/YYYY-MM-DD.md`，Obsidian 可直接打开 |
| 反思框架 —— 不止记录，还有模式识别 | ✅ 周记/月记/模式库，四层追问体系 |
| 习惯养成 —— 降低"不掉线"的门槛 | ✅ 基础习惯/可选加分/AI 协作三级体系 |

### ⚠️ 与原始愿景的差距

| 期望 | 现状 | 差距 |
|---|---|---|
| "打开就能聊，聊完日记就写好了" | AI 教练和日记页面是**分离的** —— 聊天在 `/coach`，日记在 `/today` | **核心体验断裂**：聊完的内容不会自动变成日记 |
| "比写日记启动成本低得多" | 当前 `/today` 仍需要用户面对表单填写多个字段 | 模板虽然完整，但对只想"说两句"的人来说**门槛还是高** |
| "有什么想法立刻和 AI 说" | 只有文字输入，没有语音输入 | **缺少语音入口**，不能真正做到"随时说" |
| "AI 纠正不合理的想法" | AI 提示词有 CBT 框架，但未做专门的认知矫正训练 | **模式识别停留在表面**，不能做深度的认知重构 |
| "散落的想法→结构化日记" | AI 复盘可以做，但流式输出是 raw text，未格式化存入日记 | AI 输出**没有自动填充到日记模板**的对应字段 |

### 下一步优先级

1. **🔴 打通聊天与日记** —— AI 教练对话结束后，一键生成结构化日记条目（自动填充到 `/today`）
2. **🔴 降低启动门槛** —— 首页增加"快速聊两句"入口，聊完即走，后台自动整理
3. **🟡 语音输入** —— 移动端优先，浏览器 Speech-to-Text API
4. **🟡 AI 自动填充日记字段** —— AI 复盘结果结构化解析后填入模板各字段（情绪、事实、发现、模式）

## 功能

### 已实现
- ✅ 完整的每日日记模板（今日任务、快速收纳、自由书写、晚间复盘、明日交接、习惯养成）
- ✅ AI 晚间复盘助手（DeepSeek API 驱动，流式输出）
- ✅ AI 教练对话（引导式日记写作，一次只问一个问题）
- ✅ 本地 Markdown 文件存储（数据主权，可被 Obsidian/VSCode 直接打开）
- ✅ 死亡倒计时（Memento Mori）
- ✅ 往年今日（On This Day）
- ✅ 周记 / 月记概览
- ✅ 模式库（自动从复盘中提取反复出现的模式）
- ✅ 任务队列（GTD 风格，与日记桥接）
- ✅ 日历视图
- ✅ 全文搜索
- ✅ 深色/浅色模式
- ✅ 移动端响应式

### 规划中
- ⏳ AI 模式自动识别（跨日记的模式提取）
- ⏳ 数据导出（Markdown zip）
- ⏳ 日记连续天数（streak）
- ⏳ 语音输入

## 技术栈

| 层 | 选择 |
|---|---|
| 框架 | Next.js 16 (App Router) |
| 语言 | TypeScript |
| 样式 | Tailwind CSS + shadcn/ui |
| 编辑器 | TipTap (ProseMirror) |
| 存储 | 本地 Markdown 文件 + SQLite 索引 |
| AI | DeepSeek API (OpenAI 兼容) |
| 部署 | Vercel |

## 快速开始

```bash
# 安装依赖
npm install

# 配置 API Key
cp .env.example .env.local
# 编辑 .env.local，填入 DeepSeek API Key

# 填充演示数据（可选）
npx tsx scripts/seed.ts

# 启动开发服务器
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000) 查看。

## 对比市场产品

### vs 浮墨笔记（Flomo）
| 维度 | Flomo | Miroir |
|---|---|---|
| 核心理念 | 快速捕获一切想法 | 深度处理重要想法 |
| 组织方式 | 标签 + 搜索 | 时间线 + 模式库 + AI 联结 |
| AI 角色 | 无（或基础摘要） | 教练式追问 + 模式识别 |
| 反思框架 | 无（用户自建） | 内置四层追问 + 周/月复盘 |
| 数据格式 | 专有 | 开放 Markdown |

### vs 印象笔记（Evernote）
| 维度 | Evernote | Miroir |
|---|---|---|
| 定位 | 通用笔记库 | 专注日记与反思 |
| 复杂度 | 功能臃肿 | 极简，只做一件事 |
| AI | 通用搜索/摘要 | 深度个性化教练 |
| 数据格式 | 专有 | 开放 Markdown |
